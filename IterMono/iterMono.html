Programa Iterativo

Características:

  • Estruturas de controle explícitas: Se, Enquanto, Até
  • Sem desvios incondicionais: Não usa go to
  • Hierarquia clara: Estruturas bem definidas
  • Mais legível: Fácil de entender e manter

Programa Monolítico

Instruções rotuladas com desvios:

Características:

  • Instruções rotuladas: Cada instrução tem um rótulo
  • Desvios explícitos: Usa go to para controle de fluxo
  • Fluxo linear: Segue de cima para baixo com desvios
  • Próximo da máquina: Similar a assembly/bytecode

Como a Conversão Funciona

Padrões de Conversão:

  • Enquanto T faça D: Teste T → Se verdadeiro, execute D e volte ao teste
  • Até T faça D: Execute D → Teste T → Se falso, volte a D
  • Se T então D1 senão D2: Teste T → Se verdadeiro, execute D1; senão, execute D2

Equivalência: Iterativo ↔ Monolítico ↔ Recursivo
